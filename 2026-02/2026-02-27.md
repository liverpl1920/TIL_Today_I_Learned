## やったこと
### その他
- GitHub Copilotのagentファイルにおける `model` フィールドについて調査
	- 「GitHub Copilotのagentファイルには `model` フィールドの仕様がなく、Copilot側で認識されないメタデータ」という情報が**誤り**であることを確認した
	- GitHub CopilotのプロンプトファイルおよびAgentファイル（`.prompt.md`）は、YAMLフロントマターで `model` フィールドをサポートしている
		- 参考: https://code.visualstudio.com/docs/copilot/customization/prompt-files
		- 参考: https://docs.github.com/en/copilot/tutorials/customization-library/prompt-files
	- **正しい仕様**
		- `.prompt.md` ファイルのYAMLフロントマターで使用できる主なフィールド
			- `mode`: プロンプトの動作モード（`ask`, `edit`, `agent` など）
			- `model`: 使用するAIモデルの指定（省略時はVS Code上で選択中のモデルが使用される）
			- `tools`: エージェントが使用できるツールのリスト
			- `description`: プロンプト/エージェントの説明
		- `model` フィールドの使用例:
			```markdown
			---
			mode: agent
			model: gpt-4.1
			tools:
			  - codebase
			  - editFiles
			description: コードレビューを行うエージェント
			---
			
			# コードレビューエージェント
			...
			```
		- `model` に指定できる値の例:
			- `gpt-4.1`
			- `claude-3.5-sonnet`
			- `gemini-2.5-pro`
			- ※ VS Codeのモデルピッカーに表示されるモデル名と一致する値を指定する
		- 指定したモデルが利用できない場合、Copilotはデフォルト（現在選択中）のモデルにフォールバックする
	- Claude Codeのagentファイルと比較
		- Claude Codeのagentファイルでは `model: inherit` という値が使われる（親のモデルを継承）
		- GitHub Copilotでは `model` に具体的なモデル名を指定する形式が採られている
